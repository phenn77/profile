Information

e-Procurement system is a web based application which contain the process of creating a request for items, asking for quotations from vendors, choose a winner for the suitable price from vendors, creating purchase order, and receiving items, with additional of creating invoices and payments for documentation.

Application is made with :

Sidebar #2

Description

Create Master Item

Description

Form to create an item.

Create Master Vendor

Description

Form to create a vendor.

Request Document

Description

A request document made by users which contain all the items they need.

Purchase Request Document

Description

Purchase Request (PR) Document is created from Request Document's data after the document has been approved by the superiors with changes on the item quantity.

Outstanding Items

Description

Items from all PR Documents will be grouped here for easy information gathering.

RFQ Document

Description

A RFQ (Request For Quotation) Document is where all the items from each warehouse will be grouped and created a document.

In this document, the user will insert the vendors who will participate in the procurement process.

Create Quotation

Description

Quotation can be inputted by either the vendor directly or by the users who are responsible on the document.

They may input as many quotations as they want as long the deadline is not over yet.

View Quotation

Description

Quotation Document's view after been inputted.

Winner Selection Form

Description

In this document, user will have to choose between vendors for better aspects on the item needed.

If the vendor is chosen, the user then need to assign the quantity taken from the vendor to each PR (Purchase Request) Document.

PO Document

Description

After Winner Selection Document is created, user then need to create a Purchase Order (PO) Document, which will be sent to the vendor.

In this document, the vendor will accept the document and must send the quantity of items listed on the document to the address respectively.

Receive Report Document

Description

A Receive Report which will be created when items have been sent by vendor.

Assign Item from Receive Report

Description

Items that have arrived to the user will be assigned to each individual PR Document.

Invoice Document

Description

When all the items are accepted, user will create an invoice to be given to financial division.

Payment Document

Description

This document is made just for an evidence that the procurement system has finished and the payment has been done.

Dashboard #1

Description

The dashboard is filled with Pie Chart for monthly Request / PR Document and Bar Graph for yearly quantity of Request / PR Document created.

Dashboard #2

Description

The dashboard has a table which will show the total Purchase Order Document and its price by monthly.

Print Document

Description

As for report, there are 2 types of report that are available for users. They are PDF type and CSV type, this report has supported for all kind of documents above.
